# UI - Re-run UI Research

Re-run UI research, update UI-RESEARCH.md, add gaps to fix queue.

Load and execute agents/meta/ui-researcher.md.

Process:
1. Determine app type from project-profile.md
2. Research current UI patterns for this app type (2024-2025 best practices)
3. Extract: design era, key patterns, component inventory, design tokens
4. Write/update UI-RESEARCH.md

5. Compare current implementation vs research findings:
   - Read components from src/components/
   - Check against UI-RESEARCH.md standards
   - Flag gaps (missing patterns, outdated approaches, accessibility issues)

6. Add gaps to FIX-QUEUE.md as P2 items

Report:
- Design era identified
- Key patterns for this app type
- [N] gaps found and added to queue

Purpose: Keep UI patterns current with latest best practices. Run when you want to audit or update design standards.
