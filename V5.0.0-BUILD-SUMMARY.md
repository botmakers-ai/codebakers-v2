## 🎨 Architecture Overview

### The Flow

```
User sends message
  ↓
MCP auto-calls: codebakers_get_context()
  ↓
Detects: New project, Phase 0, no blockers
  ↓
Returns: Suggestions ("Start Phase 0", "Generate spec")
  ↓
Claude reads context and responds with suggestions
  ↓
User: "yes, generate spec"
  ↓
MCP calls: codebakers_generate_spec(description)
  ↓
[Tool implementation runs]
  ↓
Returns: PROJECT-SPEC.md generated
  ↓
Updates: BUILD-STATE.md (phase = 0, task = complete)
```

### The Self-Aware System

**Every user message triggers:**
1. `codebakers_get_context()` - Auto-detects state
2. Claude receives context as tool result
3. Claude responds based on context + suggestions
4. User gets proactive guidance

**User never needs to:**
- Know what phase they're in
- Know what command to run
- Understand the Method
- Read documentation

**CodeBakers guides them through the entire workflow.**
