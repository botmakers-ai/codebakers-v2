# UI Smoke Test Generator

## Role

Automatically generates and runs three tiers of Playwright smoke tests from the project's route structure and PROJECT-SPEC.md. Ensures every page renders, every form submits without crashing, and every critical user path works end to end — before any handoff or deploy.

**Hard rule:** No feature agent output is considered "done" until Tier 1 and Tier 2 smoke tests for that feature are written and passing against the built app.

## When to Use

- After every feature agent completes (conductor triggers automatically)
- When a new route or page is added
- Before any deploy or launch
- When asked to "write smoke tests" or "test all pages"
- During a pre-launch audit

## Trigger (Automatic)

The conductor loads this agent automatically after every 3rd build agent completes. The smoke suite must pass before the next build agent starts.

---

## Three Test Tiers

### Tier 1 — Build Smoke Suite (Zero developer input required)

Generated from the route structure (`app/` directory scan). Tests that the app is alive.

```typescript
// tests/e2e/smoke-build.spec.ts
// AUTO-GENERATED by ui-smoke agent — do not edit by hand

import { test, expect } from '@playwright/test';

// Every route in app/ must:
// 1. Render without 500/404 error
// 2. Produce no console errors on load
// 3. Load in under 3 seconds

const PUBLIC_ROUTES = ['/login', '/signup', '/', '/pricing'];
const AUTH_ROUTES = ['/dashboard', '/settings', '/profile'];

test.describe('Tier 1: Build Smoke', () => {
  test.describe('Public routes', () => {
    for (const route of PUBLIC_ROUTES) {
      test(`${route} renders without errors`, async ({ page }) => {
        const errors: string[] = [];
        page.on('console', msg => { if (msg.type() === 'error') errors.push(msg.text()); });
        page.on('pageerror', err => errors.push(err.message));

        const res = await page.goto(route);
        expect(res?.status()).not.toBe(500);
        expect(res?.status()).not.toBe(404);
        expect(errors.filter(e => !e.includes('favicon'))).toHaveLength(0);
      });
    }
  });

  test.describe('Auth redirect', () => {
    for (const route of AUTH_ROUTES) {
      test(`${route} redirects unauthenticated user to /login`, async ({ page }) => {
        await page.goto(route);
        await expect(page).toHaveURL(/\/(login|signin|auth)/);
      });
    }
  });

  test('Login page loads and shows form', async ({ page }) => {
    await page.goto('/login');
    await expect(page.getByTestId('email-input')).toBeVisible();
    await expect(page.getByTestId('submit-button')).toBeVisible();
  });
});
```

### Tier 2 — Interaction Smoke Suite (Generated from PROJECT-SPEC.md entities)

Every form submits without crashing. Every button responds. Every modal opens and closes.

```typescript
// tests/e2e/smoke-interactions.spec.ts

import { test, expect } from '@playwright/test';

test.describe('Tier 2: Interaction Smoke', () => {
  // Login first
  test.use({ storageState: 'tests/e2e/.auth/user.json' });

  test('All forms submit without 500 error', async ({ page }) => {
    // For each form in the app:
    // 1. Fill required fields with valid test data
    // 2. Submit
    // 3. Expect: success state OR validation error — NOT a crash/500
  });

  test('All modals open and close', async ({ page }) => {
    // For each trigger button for a modal:
    // 1. Click trigger
    // 2. Expect modal visible
    // 3. Click close/cancel
    // 4. Expect modal gone
  });

  test('All data tables render or show empty state', async ({ page }) => {
    await page.goto('/dashboard');
    // Tables must show either data rows OR an explicit empty state message
    // A blank white area = bug
    const tables = page.locator('[data-testid$="-table"]');
    const count = await tables.count();
    for (let i = 0; i < count; i++) {
      await expect(tables.nth(i)).not.toBeEmpty();
    }
  });
});
```

### Tier 3 — Critical Path Suite (Generated from Gate 2 state changes in PROJECT-SPEC.md)

The core user workflow must work end to end.

```typescript
// tests/e2e/smoke-critical-path.spec.ts

import { test, expect } from '@playwright/test';

test.describe('Tier 3: Critical Path', () => {
  test('Full signup → onboard → core action → verify result', async ({ page }) => {
    // 1. Sign up with fresh email
    await page.goto('/signup');
    const email = `test-${Date.now()}@example.com`;
    await page.getByTestId('email-input').fill(email);
    await page.getByTestId('password-input').fill('TestPassword123!');
    await page.getByTestId('signup-button').click();

    // 2. Complete onboarding (if exists)
    if (await page.getByTestId('onboarding-step-1').isVisible().catch(() => false)) {
      // complete onboarding steps
    }

    // 3. Execute core action (from Gate 2 STATE CHANGES in PROJECT-SPEC.md)
    // e.g., create a project, upload a document, send a message

    // 4. Verify result appears in UI
  });

  test('Auth flow: login → dashboard → logout', async ({ page }) => {
    await page.goto('/login');
    await page.getByTestId('email-input').fill(process.env.TEST_USER_EMAIL!);
    await page.getByTestId('password-input').fill(process.env.TEST_USER_PASSWORD!);
    await page.getByTestId('submit-button').click();
    await expect(page).toHaveURL('/dashboard');

    await page.getByTestId('logout-button').click();
    await expect(page).toHaveURL(/\/(login|\/)/);
  });
});
```

## How to Generate Tests From Project Structure

```bash
# 1. Scan route structure
find src/app -name "page.tsx" | sed 's|src/app||;s|/page.tsx||' | sort

# 2. Find all data-testid attributes in the project
grep -rn "data-testid=" --include="*.tsx" src/ | grep -o '"[^"]*"' | sort -u

# 3. Find all forms
grep -rn "<form\|<Form" --include="*.tsx" src/ -l

# 4. Find all modals/dialogs
grep -rn "Dialog\|Modal\|Sheet" --include="*.tsx" src/ -l
```

Use this output to populate the test files above. Every route found → add to PUBLIC_ROUTES or AUTH_ROUTES. Every form found → add an interaction smoke test.

## Playwright Config — V3 Standard

```typescript
// playwright.config.ts
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests/e2e',
  fullyParallel: false,  // Run serially — built app can't handle parallel load locally
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 1 : 0,
  reporter: [['html', { outputFolder: 'tests/e2e/report' }], ['list']],

  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
    video: 'on-first-retry',
  },

  projects: [
    // Setup: create auth session once, reuse across all tests
    {
      name: 'setup',
      testMatch: '**/auth.setup.ts',
    },
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
      dependencies: ['setup'],
    },
  ],

  // ✅ V3: Run against BUILT app — not dev server
  webServer: {
    command: 'pnpm build && pnpm start',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    timeout: 180 * 1000,
  },
});
```

## Auth Setup Fixture (Reuse Session Across Tests)

```typescript
// tests/e2e/auth.setup.ts
import { test as setup, expect } from '@playwright/test';
import path from 'path';

const authFile = path.join(__dirname, '.auth/user.json');

setup('authenticate', async ({ page }) => {
  await page.goto('/login');
  await page.getByTestId('email-input').fill(process.env.TEST_USER_EMAIL!);
  await page.getByTestId('password-input').fill(process.env.TEST_USER_PASSWORD!);
  await page.getByTestId('submit-button').click();
  await expect(page).toHaveURL('/dashboard');

  // Save session state — all tests in authenticated suite reuse this
  await page.context().storageState({ path: authFile });
});
```

## What This Catches That Standard Dev Testing Misses

| Issue | How Smoke Tests Catch It |
|-------|--------------------------|
| Broken Next.js router (Sentry conflict) | Tier 1: Navigation to any route fails on day 1 |
| ESLint build errors blocking build | `pnpm build` step fails before tests even run |
| Auth redirect misconfiguration | Tier 1: Auth routes don't redirect correctly |
| Form submitting to nonexistent endpoint | Tier 2: Form submit returns 404/500 |
| Modal open but close button broken | Tier 2: Modal close test fails |
| Empty data table (no empty state) | Tier 2: Table empty check fails |
| Signup flow partially broken | Tier 3: Critical path fails at step 2 |
| Console errors on page load | Tier 1: Console error listener captures them |
