4. **codebakers_analyze_mockups**
   - Location: `src/tools/analyze-mockups.ts` (600 lines)
   - Purpose: Deep mockup analysis (data extraction, not abstract planning)
   - Input: refs/design/ folder
   - Output: MOCK-ANALYSIS.md with:

5. **codebakers_generate_schema**
   - Location: `src/tools/generate-schema.ts` (500 lines)
   - Purpose: SQL schema generation from mockup analysis
   - Input: MOCK-ANALYSIS.md
   - Output: SCHEMA.sql with:
     - Tables derived from UI requirements
     - Foreign keys (relationships)
     - Indexes (performance)
     - RLS policies (security)
   - Stack: PostgreSQL + Supabase
   - Example: User entity → `users` table with auth integration

6. **codebakers_map_dependencies** ⭐ **USER'S CRITICAL ENHANCEMENT**
   - Location: `src/tools/map-dependencies.ts` (600 lines)
   - Purpose: **Comprehensive dependency mapping from moment 1**
   - Maps:
     - **Read Dependencies:** component → stores → queries
     - **Write Dependencies:** mutation → database → ALL affected stores
     - **Store Connections:** store-to-store links
     - **Cascade Effects:** FK triggers, computed values
     - **Critical Paths:** Mutations affecting 3+ stores (high-risk)
   - Output: DEPENDENCY-MAP.md with complete graph
   - **Why Critical:** Prevents #1 bug class (mutations update DB but leave UI stale)
   - Example: Delete message → updates inbox store + thread store + unread count + folder metadata

7. **codebakers_generate_store_contracts**
   - Location: `src/tools/generate-store-contracts.ts` (430 lines)
   - Purpose: Type-safe store contracts
   - Input: SCHEMA.sql + DEPENDENCY-MAP.md
   - Output: STORE-CONTRACTS.md with:
     - TypeScript interfaces for all entities
     - Store interfaces with CRUD methods
     - State shape definitions
     - Method signatures
   - Example: Message entity → MessageStore interface with `deleteMessage(id)` method

### **Build Enforcement** (3)

9. **codebakers_enforce_feature**
   - Location: `src/tools/enforce-feature.ts` (460 lines)
   - Purpose: **FULL atomic unit protocol enforcement (no shortcuts)**
   - Trigger: `@feature [description]` or `// [description]`
   - Enforces:
     1. Context loading (BRAIN.md, dep map, error log) - MANDATORY
     2. Error Sniffer scan (all patterns checked) - MANDATORY
     3. Atomic unit declaration (in FIX-QUEUE.md BEFORE code) - MANDATORY
     4. UNIT-PROGRESS.md creation (crash recovery) - MANDATORY
     5. All 8 steps (schema → API → store → UI → states → tests → mobile → gate) - MANDATORY
     6. BUILD-LOG.md update after EVERY step - MANDATORY
     7. Gate check (all items must pass) - MANDATORY
     8. Atomic commit format: `feat(atomic): [name] — gate passed [N/N checks]` - MANDATORY
   - Output: Complete 8-step enforcement plan with checklists
   - Example: `@feature delete account` → generates full checklist, blocks shortcuts

10. **codebakers_check_scope**
    - Location: `src/tools/check-scope.ts` (299 lines)
    - Purpose: Scope verification (prevents scope creep)
    - Input: Feature description
    - Process:
      - Extracts entities from PROJECT-SPEC.md Gate 1
      - Extracts state changes from PROJECT-SPEC.md Gate 2
      - Matches feature description against spec
      - Returns confidence level (high/medium/low/none)
    - Output: Scope verification report with recommendation
    - **Blocks if scope creep:** Feature not in spec → MUST choose:
      1. Spec amendment (add to spec)
      2. Descope (remove from request)
      3. Future iteration (defer)
    - Example: User requests "AI chatbot" but spec has no AI entities → BLOCKED with options

11. **codebakers_fix_commit**
    - Location: `src/tools/fix-commit.ts` (264 lines)
    - Purpose: Git commit violation detection + auto-fix
    - Checks:
      - Commit message format (atomic format required)
      - Missing .codebakers/ files in commit
      - Missing lock files (package-lock.json when package.json changes)
    - Output: Violation report + fix commands
    - **Safe amend rules:**
      - Only amend if NOT pushed
      - Only amend if author is current user
      - Never amend other developers' commits
    - Example: Commit missing BUILD-LOG.md → generates `git add .codebakers/ && git commit --amend --no-edit`

## Installation & Setup

### **1. Install MCP Server**

```bash
cd codebakers-mcp-server
npm install
npm run build
```

### **2. Configure Claude Desktop**

Add to `~/Library/Application Support/Claude/claude_desktop_config.json` (Mac) or `%APPDATA%\Claude\claude_desktop_config.json` (Windows):

```json
{
  "mcpServers": {
    "codebakers": {
      "command": "node",
      "args": ["/absolute/path/to/codebakers-mcp-server/dist/index.js"],
      "env": {}
    }
  }
}
```

### **3. Restart Claude Desktop**

Tools will appear in tool palette with `codebakers_` prefix.


## User's Critical Enhancement

**Request:** "Best possible dependency mapping from moment 1"

**Implementation:** `codebakers_map_dependencies` (600 lines)

**What it does:**
- Maps ALL dependencies BEFORE building (not after bugs appear)
- Generates complete graph of:
  - Component → Store → Query paths
  - Mutation → Database → Store update cascades
  - Store-to-store connections
  - Foreign key triggers
  - Computed value dependencies
  - Critical paths (mutations affecting 3+ stores)

**Impact:**
- Prevents #1 bug class: "Mutation updates DB but leaves UI stale"
- Example: Delete message → system knows to update:
  - `messageStore.messages` (remove item)
  - `threadStore.threads` (update message count)
  - `folderStore.metadata` (decrement unread if was unread)
  - `inboxStore.unreadCount` (global counter)

**Before:** Developer forgets 1-2 stores → UI shows stale data
**After:** Dependency map lists ALL stores → impossible to miss
